Sojoy Scribbles

A Whimsical Sticker Shop: E-Commerce UX/UI Design & Front-End Development

Timeline

Spring 2025

Role

UX/UI Designer & Front-End Developer

Team

Inayat Inayat, Md. Kashfi Or Rashid Pranta

Tools

HTML5, CSS, JavaScript, GitHub, VS Code

Project Overview

Sojoy Scribbles is a playful, anime-inspired online sticker store designed to deliver a joyful shopping experience through hand-drawn artwork and clean, accessible UI. The website allows users to browse, customize, and purchase creative sticker products while emphasizing ease of navigation and visual storytelling.

Our goal was to create a fully responsive, multi-page e-commerce site that reflects our brand personality — bright, expressive, and user-friendly — while meeting accessibility and interaction standards.

Design Concept

"Crafting joy, one scribble at a time"

We wanted each page to feel personal and handcrafted — echoing the unique nature of the stickers themselves.

The visual system uses:

  • A soft pastel palette and friendly typography for warmth and approachability
  • Illustrated icons and imagery for a cohesive art-centric identity
  • Grid-based layouts to maintain balance and readability across pages

I contributed to the layout design, page structuring, and interaction styling across all major pages — ensuring consistency between the homepage, product listing, and checkout experience.

Site Architecture

The website includes five major pages:

Home Page

Introduces the brand with a hero section, artist story, and message form

Hero Section: Engaging tagline and CTA for browsing stickers

About Section: Personal narrative and anime-inspired imagery for relatability

Products Page

Displays sticker categories (Limited Edition & New Arrivals) with add-to-cart functionality

Product Grid: Visual-first display with hover animations and clear pricing

Cart Page

Shows selected items with dynamic quantity controls and order summary

Cart Interaction: Dynamic quantity controls and live order summary

Billing Page

Collects user shipping and payment details in a clean, structured layout

Billing Form: Structured, intuitive input fields with clear visual hierarchy

Order Confirmation Page

Confirms purchases with delivery details and order summary

Confirmation Page: Friendly, trust-building order summary

All pages utilize a shared header/footer system for consistency and ease of navigation.

My Contributions

As the team leader, I handled both design and implementation tasks:

Visual & Layout Design

Designed page structures using a responsive grid system (grid.css) and ensured visual consistency across all breakpoints.

Front-End Development

Implemented HTML5 semantic structure and CSS3 styling for all main pages (index.html, product.html, billing.html, cart.html, and order-confirmation.html).

Accessibility & Responsiveness

Optimized buttons, labels, and form fields with ARIA standards and mobile-friendly layouts.

Interactivity

Integrated simple JavaScript behaviors (like quantity adjustment and form validation) to enhance usability.

Testing & Debugging

Conducted browser and device testing to ensure cross-platform consistency.

Development Process

Our workflow was collaborative and iterative:

  • Setup: Repository initialized with .gitignore and .gitattributes for clean version control
  • Division of Tasks:
    • Inayat: All pages design, layout and interactive features, handling VS Code
    • Kashfi: Repository management, CSS setup, and documentation
  • Version Control: We worked on separate branches (addingIndex, updatingReadme) and merged updates after code review
  • Final Testing: Verified layout integrity, image optimization, and checkout flow before submission

Design Features

Product Catalog

A clean, organized product page showcasing sticker collections with category filters and seamless navigation. The layout prioritizes visual discovery while maintaining easy access to product details and pricing.

Interactive Product Grid

Visual-first display featuring hover animations that reveal product information. Each card includes clear pricing, product names, and smooth transitions that enhance the browsing experience without overwhelming the user.

Streamlined Checkout

A user-friendly billing interface with clearly structured form fields, visual hierarchy, and intuitive input validation. The design reduces checkout friction by grouping related information and providing real-time feedback.

Reflection

This project deepened my understanding of web design systems, user flow consistency, and front-end accessibility practices.

Working collaboratively strengthened my ability to translate design principles into code while balancing aesthetic expression with technical precision.

Future Iterations

  • Local storage-based cart persistence
  • Light/Dark mode toggle for inclusive design
