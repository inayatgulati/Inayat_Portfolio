Chemtrails—

All-in-one Travel Planning App

Project Overview

Chemtrails is a mobile travel app designed to help users discover destinations, plan trips, and capture travel memories in one place. Instead of switching between multiple apps for exploration, organization, and journaling, Chemtrails brings the entire travel journey into a single, cohesive experience.

Role

UX/UI Designer & Front and Back-End Developer

Team

Inayat Inayat, Shin Thant Thant

Tools

React Native, Firebase, Firestore, Google Places API, Figma

Timeline

Fall 2025

Problem

Travel planning is fragmented. Users often rely on separate apps to search places, save ideas, plan schedules, and document experiences—making the process overwhelming and disconnected.

Solution

Chemtrails unifies exploration, planning, and reflection into one intuitive mobile experience. Users can search places, save destinations into folders, schedule visits using a calendar, and journal memories with photos—all within the same app.

Goal

Design a unified mobile travel experience that reduces cognitive load by seamlessly connecting discovery, trip planning, and reflection within a single, intuitive interface.

Design Concept

Calm · Exploratory · Personal

Chemtrails is designed to feel like a travel companion, not a task manager. The interface utilizes soft colour tones, a clear hierarchy, and generous spacing to minimize cognitive load and foster a relaxed exploration experience. Content—places, images, and memories—remains the primary focus, while the UI stays intentionally minimal and supportive.

Color Palette

The color palette for Chemtrails is inspired by travel, nature, and movement, with an emphasis on calmness and clarity. Soft blues and muted greens are used as primary colors to evoke a sense of openness, reliability, and exploration. Neutral tones support backgrounds and secondary surfaces, helping content such as place images, maps, and journal photos remain the focal point. Accent colors are used sparingly for primary actions (such as saving a place or adding a journal entry) to guide user attention without overwhelming the interface.

UX rationale: A restrained palette reduces visual fatigue, supports long browsing sessions, and lowers cognitive load during planning-heavy tasks.

Key Features

Explore Destinations

Users can search for destinations using real-time Google Places data, view ratings, reviews, photos, and directions, and save locations instantly to their wishlist.

Wishlist & Calendar Planning

Saved places can be organized into custom folders and assigned visit dates through a built-in calendar, transforming inspiration into actionable plans.

Home Dashboard

A personalized dashboard featuring dynamic greetings, rotating travel quotes, upcoming plans, and featured destinations creates a welcoming starting point.

Travel Journal

Users can create, edit, and delete journal entries with multiple photos, allowing them to document trips as personal narratives rather than checklists.

Profile and Stats

The profile page tracks user identity and automatically generated travel statistics such as cities visited and trips journaled.

Technical Overview

Frontend: React Native (Expo)

Backend: Firebase Authentication, Firestore

Media Storage: Firebase Storage

APIs: Google Places API

Navigation: Bottom tab navigation

Challenges and Learning

This was my first full mobile app built with React Native. I encountered challenges related to navigation state, image uploads, API error handling, and calendar logic. Through documentation-driven learning, debugging, and iteration, I gained confidence in building scalable, multi-screen mobile applications and translating UX designs into functional products.

Outcome

Chemtrails resulted in a fully functional mobile application that supports real user data, media uploads, and external APIs. The project strengthened my skills in UX thinking, mobile development, and problem-solving, and demonstrates my ability to design and build end-to-end digital products.
